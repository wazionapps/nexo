NEXO 7.22.0 — fast MCP gates and Entity Dossier

Published 2026-05-17. Minor release over v7.21.0.

Why this release exists

Desktop conversations should not wait for optional memory analysis, local classifiers, or a busy SQLite writer before the first useful answer. v7.22.0 keeps the protocol obligations, but moves heavy work and uncertain writes out of the visible chat path.

What changed

  • Fast heartbeat path. Desktop-managed sessions no longer load Local Context, drive/adaptive analysis or semantic correction classifiers from normal heartbeat calls unless explicitly forced for diagnosis.
  • Durable MCP write queue. Interactive writes can be accepted outside SQLite, then drained by a runtime writer with clear queued, committed, retrying and dead-letter states.
  • Compliance state. Brain exposes heartbeat, diary, learning and clean-close obligations so Desktop can block before the next visible turn or close path when evidence is missing.
  • Essential diary write tool. Session diary writing is available even when dynamic plugin loading is disabled, preventing close/archive flows from depending on an optional plugin surface.
  • Entity Dossier. Local Context can resolve entity aliases, extract open-domain facts, aggregate linked documents and return evidence references without changing the existing semantic search path.
  • Indexer cooperation. Local Memory can persist desired speed under load and react to Desktop foreground state so active conversations do not compete with extreme indexing.
