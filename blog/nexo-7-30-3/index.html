NEXO 7.30.3 - release closeout backup protection

v7.30.3 is a focused Brain patch over v7.30.2. It keeps the Deep Sleep release scope intact and fixes the final local closeout path that could fail after publication when technical backup pruning removed a freshly created pre-autoupdate-* snapshot before validation checked nexo.db.

paths.finalize_backup_snapshot() now passes the snapshot being finalized as a protected path to the runtime backup pruner. Disk-pressure cleanup can still rotate older technical snapshots, but it cannot delete the backup the updater is about to validate or use for rollback.

The release final-closeout verifier also resolves the active runtime database through the canonical path helper. This keeps protocol evidence checks aligned with the modern ~/.nexo/runtime/data/nexo.db layout while preserving legacy compatibility where needed.

Regression tests cover protected emergency pruning, validated backup survival through finalize/prune, and protocol closeout against the modern runtime database path.
