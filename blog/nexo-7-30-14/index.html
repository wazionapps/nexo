NEXO 7.30.14: support tickets and capability discovery

June 5, 2026. This patch tightens the agent-facing product contract used by Desktop-managed assistants.

What changed

  • Agents now have explicit tools to list, read, and create real authenticated support tickets.
  • The system catalog advertises support tickets, provider proxy platforms/models, NEXO Credits Cloud/Edge, managed Email NEXO, and protocol cards.
  • After a task is closed, task-open reminders wait for a new visible user message instead of rearming from background activity.
  • Internal self-audit followups are marked as agent-owned internal items, and memory-observation drift is watched by a scheduled runtime job.

Why it matters

Desktop agents need to route real customer incidents to support tickets, discover the backend capabilities they can use, and avoid reopening protocol loops after a task has already closed. 7.30.14 makes those behaviors explicit in Brain instead of relying on prompt memory.
