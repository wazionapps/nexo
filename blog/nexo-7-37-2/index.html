NEXO 7.37.2 - runtime shutdown and CI stability

v7.37.2 is a focused corrective release over v7.37.1. It ships the keepalive/SQLite shutdown hardening found during the coordinated Brain/Desktop release audit and keeps the public package aligned with the now-green Brain main branch.

Keepalive writers stop before SQLite closes. The session keepalive supervisor now tracks and joins its background threads, and test setup/teardown drains those writers before closing the shared Brain DB. This removes the race that could crash the full pytest suite during shutdown.

SQLite close uses the same serialization path as writes. The serialized connection wrapper now closes under the write lock, so a late writer cannot race the underlying connection teardown.

The full test workflow has enough time to finish. The GitHub Actions test timeout is raised to 45 minutes, so slow-but-valid full-suite runs fail on real errors instead of being cancelled mid-run.
