NEXO 7.30.18: managed email identity

June 6, 2026. This patch fixes managed email signatures for secondary or clean installs with their own assistant identity.

What changed

  • nexo-send-reply.py now resolves the assistant display name from runtime calibration and profile before account metadata.
  • If no runtime identity is present, the sender account can provide a safe fallback, such as torrente@nexoagentmail.com resolving to Torrente.
  • Stale generated signatures like Nero — torrente@nexoagentmail.com are refreshed, while custom signatures remain untouched.

Why it matters

Automated operational mail should reflect the machine or account that sends it. Clean installs configured as a separate assistant no longer inherit another runtime's visible name in alert signatures.
