NEXO 7.31.9 - original symptom verification before UI release closeout

v7.31.9 is a patch release over v7.31.8. It hardens the closeout path for UI and front-end work: a release-ready claim can no longer be closed with generic "tests are green" evidence when the operator reported a visible symptom.

The closeout evidence now has to name the original reported symptom and attach observable proof from the real surface. Accepted proof includes a repro URL, headed browser or screenshot evidence, Playwright output, curl output, or relevant live logs.

This pushes agents to re-open the exact screen or flow that failed before saying a UI release is ready, reducing the pattern where a fix passes tests but the reported problem was never checked again.
