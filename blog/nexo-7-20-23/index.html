NEXO 7.20.23 — local memory sidecar status

Published 2026-05-16. Patch release over v7.20.22.

Why this release exists

Local Memory now stores its heavy file index in runtime/memory/local-context.db. v7.20.23 makes status reads use that sidecar database directly and read-only, so Desktop sees real indexing progress instead of the empty legacy main database.

What changed

  • Read-only sidecar status. Status no longer runs migrations or writes when Desktop only needs progress.
  • No false zeroes. Busy, missing-schema, invalid or unreadable sidecar states return retryable keyed errors with no fake counters.
  • Localized recovery copy. Problem payloads use message/action keys so Desktop can render Spanish and English consistently.
  • Service metadata preserved. Error paths still include service health and problem state for support diagnostics.
