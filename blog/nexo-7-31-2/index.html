NEXO 7.31.2: session continuity and resident stability

June 11, 2026. This patch release is the second half of the Phase 2 reliability program: continuity that survives quiet work, and a runtime resident that nothing can knock over by accident.

What changed

  • Sessions survive quiet work periods. Session rows used to be physically deleted after 15 quiet minutes by whichever session or cron started next — any client that spent a while in code tools lost its session mid-task and orphaned its open work. The session ID is now a durable identity: tools revive a valid SID whose row vanished, physical cleanup only happens after 24 hours, and the 15-minute window keeps governing what counts as "active" in listings.
  • Residents from different installs can no longer kill each other. Two Brain installs sharing one machine used to share one runtime-service state file and killed each other's resident in an endless ping-pong (1,314 restarts logged on one machine; every restart forced a 10-48 second cold-Brain conversation start). State files are now keyed by runtime generation: a runtime can only see — and manage — its own resident.
  • Obsolete residents retire themselves. After an update, the old resident exits cleanly once no clients remain connected; the current one never self-terminates, keeping the Brain warm and conversation starts fast.

Why it matters

Continuity is the product. An agent that forgets its session because its operator spent twenty minutes compiling, or that boots cold because two installs were fighting over a port, is an agent you cannot trust with long work. 7.31.2 turns both failure modes off at the root.
