NEXO 7.38.0 - closeout integrity and operator-facing discipline

v7.38.0 is a minor release over v7.37.4. It tightens how work is reported, verified, and closed, so long tasks stay legible to the operator and nothing is marked done without proof.

Long, opaque work now reports its own progress. When a task runs long without visible output, the protocol expects a periodic plain-language status update instead of silence, and the first response to a request leads with the decision or status rather than internal jargon.

Batch closeouts carry per-item verified state. Closing many tickets, files, or tasks at once requires each item's individual verified state, so a blanket "all done" can no longer hide an unverified item, and closeouts converge on a short, consistent operational template.

Production changes create an automatic ledger. Production mutations are detected across a much wider surface - gcloud builds/run/dns, database migrations (alembic, prisma, sequelize, knex, rails, django, artisan), cPanel, and Cloudflare DNS - and a change-log entry is expected automatically at closeout. New closure-promise, evidence-matrix, and cost-secret-sweep audit modules reconcile promises against closeouts and scan for accidental cost or secret exposure.

The correction-learning gate never blocks mid-work. A detected user correction without a persisted learning now opens a non-blocking debt that a later learning resolves, instead of trapping the agent in the middle of a task. New behavioral rules ship in shadow/log mode by default and observe before they ever gate.
