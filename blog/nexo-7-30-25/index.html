NEXO 7.30.25: maintenance self-heal diagnostics

June 8, 2026. This patch reduces false maintenance warnings for Desktop-facing installs. When maintenance is healthy, self-healing or informational state now stays out of the user's way instead of asking a non-technical user to interpret internal repair details.

What changed

  • LaunchAgent calendar checks now compare manifest weekday/weekdays schedules the same way cron sync declares them.
  • Release audit traces are classified as operator history instead of current installation health.
  • Sparse, high-coverage telemetry gaps remain informational instead of producing noisy maintenance warnings.
  • Local-index hygiene retries transient SQLite locks and treats truncated quick scans as clean when no residue is found.
  • Watchdog reads the canonical runtime/cognitive/cognitive.db path with legacy fallback and observes live long-running workers without warning.

Why it matters

Desktop users should not need to understand LaunchAgents, SQLite locks, telemetry coverage, or watchdog internals. This release keeps those maintenance details inside Brain unless a real user-actionable problem remains.
