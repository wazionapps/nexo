NEXO 7.37.1 - release hardening for Desktop-bundled Brain

v7.37.1 is a corrective release over v7.37.0 focused on the Brain snapshot that Desktop packages for new installs and updater-driven reinstalls. It keeps the transparent server self-heal and email zombie reinjection guard, then closes the remaining release-readiness gaps found in the coordinated Brain/Desktop audit.

Large indexes avoid foreground stalls. The 7.36.0 disk-reclaim line still converts small and normal legacy local-context.db files to auto_vacuum=INCREMENTAL, but v7.37.1 defers the one-time synchronous rewrite when the database is too large for a safe startup path. The default foreground cap is 128 MiB and can be tuned with NEXO_LOCAL_CONTEXT_AUTO_VACUUM_MAX_SYNC_BYTES.

The advertised MCP surface matches runtime startup. The nexo_schema_abstraction_* tools were present in the enforcement map and plugin registry, but the essential plugin startup set could leave them unloaded. The plugin is now in that essential set, so clients do not see a tool contract that the default runtime cannot serve.

Desktop compatibility payloads are tolerated. nexo_learning_list and nexo_learning_search now accept the compatibility limit key sent by Desktop-side callers while preserving Brain-owned grouping and ranking behavior.
