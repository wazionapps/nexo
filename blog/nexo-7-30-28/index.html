NEXO 7.30.28: first-hop F0.6 repair

June 8, 2026. This patch makes the F0.6 repair execute on the first upgrade from older installed versions. It routes script-conflict recovery and core/current refresh through an existing post-install hook that old updaters already invoke.

What changed

  • The existing guardian-hard-persisted hook now also runs F0.6 script-conflict recovery and same-version snapshot refresh.
  • Fresh hook execution prefers the hook list from the newly copied runtime tree, preventing future hooks from being skipped by an older parent updater.
  • The existing F0.6 scripts -> core/scripts propagation and personal-script preservation behavior remains covered.
  • Regression tests cover old-hook compatibility, conflict recovery, same-version snapshot refresh, and the active watchdog LaunchAgent path.

Why it matters

A user should not need to know that a runtime has compatibility shims, snapshots, or old updater behavior. This patch makes the update repair that state itself.
