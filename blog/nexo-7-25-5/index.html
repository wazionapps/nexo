NEXO 7.25.5 - explicit Local Memory include overrides

v7.25.5 closes the final Local Memory roots v2 gap found during release audit. Default discovery remains conservative, but an explicit user include is now treated as intentional consent to index that root even when it is normally skipped as a system, cache, dependency or noisy subtree.

This means users can include root paths such as /, drive roots, external folders, or normally skipped folders like .venv when they really want them. Manual exclusions still win, and normal fresh installs still start from safe user-content and email roots.

The patch also keeps Brain LocalIndex operator-facing messages in English, leaving Spanish/English localization to Desktop, and keeps Desktop-managed install guidance aligned with the Core-pinned Python 3.12 runtime.

nexo local-context roots add /path/to/include --json
nexo local-context exclusions add /path/to/skip --json
nexo local-context filetypes include .asd --json
