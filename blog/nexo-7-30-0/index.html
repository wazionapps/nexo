NEXO 7.30.0 - agent-facing preferences

v7.30.0 is a Brain minor release focused on making NEXO easier to operate for non-technical users. The agent can now list, explain, and change supported preferences by chat, including morning briefing content, schedule, client defaults, and selected profile settings.

The morning briefing can include weather and public news when enabled. Weather uses the location saved in Desktop or the profile, while unavailable external data is treated as a missing source instead of failing the full briefing.

Automation schedules now support selected weekdays such as 07:00 Mon-Fri or 07:00 Tue,Sat. The same contract is handled by CLI, Desktop IPC, launchd, cron, systemd, and catch-up recovery.

Claude Code defaults now move from Opus 4.7 to claude-opus-4-8 while Codex stays on the verified gpt-5.5 tiers. Existing default resonance contracts migrate safely, but custom model selections are preserved.

This release also fixes the certificate verification path used by SMTP sending, addressing cases where the morning briefing was generated but showed as a failed execution during email delivery.
