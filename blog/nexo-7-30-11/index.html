NEXO 7.30.11: installer repair baseline

June 4, 2026. This patch closes the first-update gap found during local verification of 7.30.10.

What changed

  • The installer now writes `runtime/operations/last-repair-baseline.json` after verified migration, same-version repair, and fresh install layout finalization.
  • npm `postinstall` also refreshes the baseline when the installed version already matches the package and would otherwise exit immediately.
  • Tests now pin the installer/postinstall contract alongside the packaged updater and runtime post-sync baseline paths.

Why it matters

7.30.10 fixed the updater path once new code was running, but an older installed updater could not execute the new Python baseline helper during the very first package upgrade. 7.30.11 moves that safety point into the new package installer, so old and new users get the repair marker through the normal update/install flow.
