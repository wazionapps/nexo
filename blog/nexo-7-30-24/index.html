NEXO 7.30.24: managed MCP client sync release gate

June 7, 2026. This patch closes a release blocker in managed MCP client sync: Brain-owned default MCP entries are no longer written into user clients unless the required providers have been staged and verified as healthy.

What changed

  • client_sync now runs managed MCP reconciliation with apply=True before merging Brain-owned defaults.
  • nexo_chrome_control, nexo_desktop_control, and nexo_power_control are written only when their required providers report healthy.
  • When staging fails, sync removes stale NEXO-owned managed MCP entries from JSON and TOML client configs instead of leaving broken wrappers behind.
  • User-owned MCP entries are preserved, even when they use the same server names.
  • The fix does not add an automatic npx fallback, so failed provider staging remains visible and reproducible.

Why it matters

A client config that points to an unstaged managed MCP provider can close the MCP handshake before initialization. This release makes provider staging a hard precondition for writing Brain-owned managed defaults and treats cleanup as part of the same gate.
