NEXO 7.37.0 - transparent server self-heal

v7.37.0 removes the user-visible restart gap after a Brain update. When a resident server detects that the installed code has moved to a newer version, it re-launches itself in place before continuing to serve the client.

Updates apply without a manual restart. The server preserves the live connection and waits for any in-flight tool call before re-execing, so an update can take effect without interrupting the current response. Boot-time checking also heals before the server starts serving a stale runtime.

The self-heal path fails open. Non-POSIX hosts, resident background services, re-exec errors, and the NEXO_DISABLE_SELFHEAL_REEXEC kill switch fall back to the previous safe behavior. Bounded generations and same-target guards prevent restart loops.

Already-replied emails are not resent. If a session died after sending a reply while the email was still marked processing, the monitor could later reinject that email and send a duplicate. The monitor now recognizes the already-sent reply, marks the email processed, and reports the reconciliation.
