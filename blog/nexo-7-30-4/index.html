NEXO 7.30.4 - local update completion and runner closeout

v7.30.4 is a focused Brain patch over v7.30.3. It fixes a local update closeout failure where the post-sync runtime initialization could roll back after 60 seconds even though the bounded Memory Fabric repair was healthy and completed shortly after that limit.

The updater now uses an explicit RUNTIME_POST_SYNC_TIMEOUT_SECONDS budget of 180 seconds for the database, personal schedule, skill, script, and Memory Fabric reconciliation subprocess.

It also treats nexo_stop as terminal in headless automation cycles, so followup-runner, Deep Sleep, and cron sessions can finish cleanly instead of reopening a protocol reminder loop after a no-op close.

This keeps local update completion aligned with real installed runtimes that have substantial transcript and historical diary indexes, without weakening backup validation or public release gates.
