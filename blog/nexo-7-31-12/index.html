NEXO 7.31.12 - Local Memory core hardening + offline-first installer

v7.31.12 is a patch release over v7.31.11. It hardens the Local Memory core and makes clean installs provision their Python dependencies reliably, offline-first.

Retrieval is more robust: cosine similarity is now normalized at comparison time, so ranking stays correct even if the embedding model is swapped for one that does not L2-normalize its output. Chunk ids are position-stable (keyed on version and index, never on chunk text), so re-indexing is idempotent and the entity-fact / alias dedup holds across runs instead of accumulating duplicates on every content edit.

iCloud files that macOS has evicted to the cloud (the SF_DATALESS flag) are now indexed metadata-only instead of faulting in a download from a headless daemon. The EDEADLK that a failed materialization raises is classified as "offloaded" rather than a reliability error, the silent 20-error logging cap is removed so the true failure scope is visible, and status() reports a backlog drain rate. The index DB writer also gains wal_autocheckpoint, mmap_size and cache_size PRAGMAs.

On the install side, dependencies are provisioned offline-first from a bundled wheel set with per-module venv self-repair wired into startup. A clean install no longer silently indexes PDFs, spreadsheets or emails as empty text because a parser or the embedder was missing.
