NEXO 7.38.1 - hook recovery and runtime safety follow-through

v7.38.1 is a patch release over v7.38.0. It closes the gap between source and installed runtime for the thinking-block 400 recovery path, while carrying the reviewed safety changes already present on main.

Known thinking-block 400 failures now preserve state. Managed hooks detect the client-side `thinking` / `redacted_thinking` failure pattern, write a checkpoint and diary draft, and guide the operator to run `/clear` before continuing from the saved state.

The release includes reviewed runtime safeguards. The package carries the live-context preflight, production-path write observation, deploy-close evidence gate, correction-learning follow-through, and the `cost_secret_sweep` false-positive fix that avoids cross-line secret matches.

Operator-facing behavior stays practical. The recovery path explains the next action in plain language and avoids editing installed runtime code directly; installs receive the fix through the normal packaged update route.
