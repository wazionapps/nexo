NEXO 7.30.17: F0.6 personal script import shim

June 6, 2026. This patch repairs a compatibility gap in the F0.6 runtime layout migration.

What changed

  • F0.6 now exposes nexo_personal_automation.py from personal/scripts as a core-backed compatibility shim.
  • The normal repair uses a relative symlink, so the script catalog deduplicates it against the core helper.
  • Distinct operator-owned helpers with the same filename are preserved instead of overwritten.

Why it matters

Existing personal automations often run directly from personal/scripts. Those processes only put that directory on sys.path, so bare imports to helpers promoted into core/scripts needed an explicit compatibility entry.
