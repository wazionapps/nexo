NEXO 7.30.31: Core Rules injection

June 8, 2026. This patch makes protected product Core Rules visible where agents actually need them: in the managed startup bootstrap and in task-specific pre-action checks.

What changed

  • Claude and Codex managed bootstraps now include a compact Core Rules summary from the first turn.
  • cortex/task_open now pulls relevant product_core and bootstrap_contract rules from the protected registry.
  • The full source of truth remains src/rules/core-rules.json plus the synchronized DB table core_rules.
  • Regression tests prove answer/edit tasks receive the right product and bootstrap rules.

Why it matters

NEXO should not ask for things it already knows, deny capabilities without checking, or rebuild from zero when prior work exists. This release moves those rules from stored policy into the agent's active working context.
