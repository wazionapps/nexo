NEXO 7.30.33: personal agent run state

June 8, 2026. This patch fixes a state reconciliation bug in the personal scripts registry. When an operator manually ran an agent successfully after an older scheduled cron failure, the agent view could still report the old failure.

What changed

  • list_personal_scripts() now compares the stored manual run timestamp with scheduled cron history and keeps the newest real execution.
  • nexo agents status and script-backed agent health no longer mark an agent as failing when the latest manual run succeeded.
  • A regression test covers the exact scenario: newer manual success, older cron failure, and no duplicate script or recovery layer.

Why it matters

Personal agents should reflect operational reality. If a user or another NEXO session runs an agent manually and it succeeds, the product should not keep showing stale failure state from an older schedule.
