NEXO 7.31.8 - email monitor waiting-state debt suppression

v7.31.8 is a patch release over v7.31.7. It fixes a debt-detection edge case in the email monitor: a thread that was already resolved into an intentional waiting state could still be flagged later as an unresolved commitment.

The debt scan now checks for recent waiting-state resolutions and active hot-context state before escalating commitment/action debt. If the thread is waiting on the user or a third party, the monitor suppresses the stale debt signal. If the commitment is genuinely unresolved, the existing path still flags it.

This keeps the monitor from creating false urgency on emails where the correct next action is to wait, while preserving the protection that catches real forgotten commitments.
