NEXO 7.32.0 - Cognitive OS Ola 1: connect-the-dots + reliability

v7.32.0 is a minor release over v7.31.13 and bundles the v7.31.14 critical fixes. It is the first wave of the "NEXO Cognitive OS" plan: wiring up cognitive machinery that existed in the codebase but was never connected.

The causal/provenance graph finally populates. Every evidence-backed task close now records graph edges (task → change_log as "ops:produced", and change_log → task as "causal:motivated_by"). The associative substrate that powers connect-the-dots reasoning was previously empty because the recorder had no caller; it now grows with real work. The wiring is best-effort and never blocks a close.

No more zombie workflows. Session auto-close previously reaped only protocol tasks, so durable workflow runs and goals a stale session left open lingered forever and polluted the cross-session resume surface. They are now reaped to a terminal state when their owning session is reaped.

Also included: the v7.31.14 critical fixes - the cron-fleet drift guard (a test run could no longer rewrite the operator's real LaunchAgents), deterministic spreading-activation ids, real guard_context evidence (no more fake evidence satisfying the critical gate), working auto error-to-learning capture, a fulfillable confidence-check gate, and approval-paused workflow resume.
