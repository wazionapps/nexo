NEXO 7.34.0 - Cognitive OS Ola 2: working memory + associative graph at answer time

v7.34.0 is the second wave of the "NEXO Cognitive OS" plan. It adds a working memory that stops re-doing work it just did, learns from its own mistakes, connects the dots across what it knows at answer time, and tidies its memory while you sleep.

Working memory that never goes stale. A resolution_cache fast-path short-circuits a resolution it has already done (an entity, a file, a fact) instead of re-running the full search. It is never-stale by construction: fail-closed, keyed on a content snapshot, gated by a global write watermark so any newer change invalidates it, with a 15-minute TTL; code resolutions carry a repo-map so a changed file forces a re-resolve.

It learns from its own mistakes. When a later step reveals that an earlier action was wrong, NEXO now auto-captures a learning plus a concrete prevention - high precision, no noise - so the same error is caught next time instead of waiting for you to flag it.

The associative graph connects the dots at answer time. Answering runs a bounded Personalized PageRank walk over the knowledge graph from the entities and files you mention, surfacing non-obvious related context. Anti-hub weighting stops generic high-degree nodes from dominating, it is fail-open, and results are cached per process.

Deep Sleep that rewrites, safely. A nightly phase merges duplicate and near-duplicate learnings so the rule corpus stops bloating - reversible by design (supersede/merge only, zero hard-delete), a fail-closed backup before any change, and a daily cap. A reproducible memory-recall eval bank (recall@k/MRR) ships with a baseline so retrieval regressions are caught between releases.
