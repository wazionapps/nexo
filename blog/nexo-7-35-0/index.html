NEXO 7.35.0 - Cognitive OS Ola 4: selective forget + recurring-incident diagnostic templates

v7.35.0 is the fourth wave of the "NEXO Cognitive OS" plan. It gives the brain a safe way to forget what should not be remembered, turns repeated failures into reusable diagnoses, and removes a protocol false-positive on routine local work.

Selective forget - delete what should not be remembered, and prove it is gone. Two reversible-by-design modes. hard-forget removes a leaked secret (an API key, a token) or a genuinely-wrong memory and then verifies zero survivors across every live store - all canonical databases, full-text search, the vector index, and transcripts - with secure_delete=ON so the bytes are overwritten, not just unlinked. correct-fact supersedes a wrong fact with a soft, undoable correction. This closes the long tail of incidents where a secret landed in memory and had to be scrubbed everywhere at once.

Recurring failures become reusable diagnoses. When the same failure archetype recurs across separate incidents - for example a job that exits 0 but silently did nothing - Deep Sleep distills it into a reusable diagnostic template that is primed before a matching action, so the known checks run first instead of rediscovering the failure from scratch. Strong/weak marker tiers keep benign success phrasing ("exit 0 on success", "silent deploy") from ever seeding a template: a single weak hint is ambiguous and ignored, while a strong signal (|| echo, "forgot the cron", "swallowed the error") or two weak ones are required. Injection is guidance-only and never blocks an action.

Protocol precision. Closing a local-only followup-runner is no longer mis-flagged as an external, real-world-stakes action, so routine local work stops triggering spurious closure challenges.
