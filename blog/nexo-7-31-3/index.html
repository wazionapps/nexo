NEXO 7.31.3: default Claude model back to Opus 4.8

June 11, 2026. A small operator-driven patch: the recommended Claude Code model returns to Opus 4.8, and a trio of dead hooks leaves the tree for good.

What changed

  • Default Claude Code model: Opus 4.8 with max reasoning. This reverts the brief 7.31.0 move to Fable 5. Installs riding NEXO defaults migrate back automatically (recommendation version 5, with Fable 5 recorded as a previous default); deliberately customized models stay untouched, as always. The four main resonance tiers resolve Claude Code to Opus 4.8 again, the cheap classifier tier keeps Haiku, and Codex routing is untouched.
  • Dead heartbeat-enforcement hooks removed. The trio was never wired into client settings and its output was discarded; update cleanups already retired it from installs, and the protocol nudge shaping introduced in 7.31.1 covers the need. A contract test now pins their absence.

Why it matters

Model defaults follow the operator's judgement, and the migration machinery makes those changes safe in both directions: riders of the recommended default move automatically, custom choices are never touched, and dead code stays dead.
