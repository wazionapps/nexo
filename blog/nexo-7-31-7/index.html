NEXO 7.31.7 - pre-answer evidence before state claims

v7.31.7 is a patch release over v7.31.6. It closes an obedience gap: when the user asks whether something is published, merged, sent, installed, verified, closed, running, deployed, or tied to a release/commit/server/ticket, the agent must verify first instead of answering from memory.

The pre-answer router now has a live_state_claim path. It checks continuity and evidence sources such as recent context, evidence ledger, change log, protocol tasks, workflows, project atlas, system catalog, diary, transcripts, memory, and local context. If those sources do not prove the state, Brain injects a verification-gap reminder instructing the agent to say the state is not verified yet and keep checking.

This release also promotes identity coherence to a hard/core default and adds the new hard/core R37 evidence gate to Guardian defaults and core prompts. Closeout behavior is stricter: future commitments need followups, total "no debt" claims are blocked while work remains open, release/smoke/tag work cannot close as done while still described as pending, and irreversible publication claims need specific approval evidence.
