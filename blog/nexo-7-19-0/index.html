NEXO 7.19.0 — bundle-managed updates and version-drift autoexit

Published 2026-05-12. Minor release over v7.18.1.

Why this release exists

Coordinated NEXO Desktop releases ship a frozen brain-bundle/ snapshot of Brain inside the application package. When the live server.py process kept auto-updating from npm afterwards, the bundled snapshot and the live process could drift to different versions, which then produced "Restart this MCP client" errors and stale orphan server.py processes whenever a client did not honor the restart hint.

What changed

Two cooperative changes turn the bundle into the authoritative source for the version installed on a host:

  • Auto-update kill-switch. When NEXO_BRAIN_AUTO_UPDATE is set to 0, false, off or no, startup_preflight records skipped_reason="auto_update disabled via NEXO_BRAIN_AUTO_UPDATE env var" and returns immediately without checking npm. Bundle-managed installs (NEXO Desktop) set this on spawn so the bundled Brain version stays pinned to the host application release cycle.
  • Drift autoexit. When RestartRequiredMiddleware detects installed_version != process_version, after delivering the existing mcp_restart_required payload the server schedules os._exit(75) (EX_TEMPFAIL) ~500 ms later so MCP clients respawn the server with the new code instead of leaving stale processes alive. The autoexit is idempotent and falls back to an immediate exit when no asyncio loop is running.
