NEXO 7.25.4 - Local Memory roots and file-type controls

v7.25.4 changes Local Memory's default shape: fresh installs now start from safe user-content and email roots instead of whole-disk roots. System volumes, mounted drives, network shares and root paths such as / are still available, but only when the user explicitly adds them.

Updates get a cleanup path for legacy whole-disk indexes. The roots v2 migration removes legacy system roots, keeps user content, preserves user exclusions and file-type overrides, and uses a validated backup for smaller databases. Very large local-context.db files are archived and rebuilt from lightweight configuration instead of being purged in place.

The release also adds configurable file-type rules through CLI and MCP. Core defaults read useful documents, text, email and selected code files, keep photos and media as metadata, ignore unknown/binary/transient extensions, and let users include a custom extension or exclude one they do not want indexed.

nexo local-context filetypes list --json
nexo local-context filetypes include .asd --json
nexo local-context migrate-roots-v2 --json
