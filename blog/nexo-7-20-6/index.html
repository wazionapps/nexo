NEXO 7.20.6 — local memory evidence and email coverage

Published 2026-05-13. Patch release over v7.20.5.

Why this release exists

Local Context must act like useful operational memory, not just a file index. v7.20.6 fixes the case where Brain detected an entity such as a project, client or function name but retrieved unrelated chunks, and it adds safe automatic coverage for local email sources that were too deep for the default home-root scan.

What changed

  • Entity evidence is chunk-aware. Entity hits bring the matching asset into retrieval, but chunks with direct co-occurrence in text, path or summary rank above unrelated chunks from the same long document.
  • Older entity assets remain eligible. Matched entity assets are scored even when newer unrelated chunks fill the normal recent-chunk window.
  • Email roots are automatic. Brain adds bounded local email roots for Apple Mail, Outlook, Thunderbird and NEXO email continuity instead of requiring users to configure deep paths manually.
  • Email extraction is safer and broader. `.eml`, Apple Mail `.emlx`, Outlook `.msg` best-effort, and NEXO `nexo-email.db` sent/received continuity are extracted. Large Outlook `.pst`/`.ost` stores remain inventory-only until a dedicated parser is safe enough to ship.
  • Agents see relations before acting. Pre-action context now includes local graph relation lines next to assets and evidence refs.
