NEXO 7.30.32: packaged update and automation health

June 8, 2026. This patch closes update and observability gaps found in live runtime maintenance: broken packaged npm wrappers, stale personal script backup artifacts, scheduled automations without a health signal, and managed agents writing durable notes into legacy client memory files.

What changed

  • Packaged update and doctor share the same npm toolchain probe and can repair broken NEXO-owned npm/npx wrappers before using npm.
  • Doctor can archive stale NEXO-owned personal script backup artifacts such as .bak.* files instead of reporting false duplicate-script warnings.
  • Scheduled personal automations can declare health_file, health_max_age_seconds, health_nonempty, and health_must_contain contracts so silent failures become visible.
  • Managed agent memory writes to legacy Claude/Codex project MEMORY.md paths are blocked; durable memory must go through NEXO Brain.

Why it matters

NEXO should repair update prerequisites before asking the user, detect broken scheduled work before the operator notices manually, and keep memory authority inside the shared Brain instead of legacy client files.
