Operator Docs

Fastest path from install to real use

These are the commands and concepts that matter first. Use the repo wiki for deeper reference afterwards.

1. Install

Install the runtime

Use npm once. The installer sets up the local runtime, MCP wiring, and the core background jobs.

$ npx nexo-brain
2. Launch

Start Claude Code through NEXO

Use the runtime entrypoint instead of a manual shell setup.

$ nexo chat
3. Inspect

Check runtime health

Doctor is the main operational entrypoint for health, drift, and deterministic repairs.

$ nexo doctor --tier runtime --json
Daily Operations

The commands worth remembering

Launch

nexo chat

Open Claude Code through the NEXO runtime in the current directory or a target path.

Update

nexo update

Sync the installed runtime and run the official post-sync path. Newer builds now reconcile declared personal schedules automatically.

Diagnose

nexo doctor

Inspect boot/runtime/deep health. Use --fix for deterministic repairs when the doctor says it is safe.

Automate

nexo scripts

Manage personal scripts: list, classify, reconcile, schedule, unschedule, and validate them through the supported flow.

Sleep, Wake, Power

What to expect on a laptop

NEXO now supports wake-aware recovery contracts for core jobs and managed personal scripts. The optional always_on policy enables the platform power helper for best-effort background availability, but wake recovery and catch-up remain part of the design.

Core jobs

13 core recovery-aware jobs are declared in the runtime manifest and synced by the official installer/update flow.

Personal scripts

Managed scripts can declare run_once_on_wake, catchup, restart, idempotency, and catch-up windows.

Power helper

On macOS, NEXO uses native caffeinate. This helps availability, but it is not sold as a blanket closed-lid guarantee.
