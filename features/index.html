The Runtime Layer

What the public site should make obvious

NEXO is not just a memory plugin. It is the operational layer around your agent.

Startup preflight

Before interactive work, NEXO can run safe local migrations and health preparation so the runtime starts from a sane state.

Doctor diagnostics

nexo doctor inspects boot/runtime/deep health and can apply deterministic fixes for drift, schedules, and runtime mismatches.

Recovery-aware jobs

Core jobs declare catch-up or restart contracts instead of assuming the host was awake at the perfect minute.

First-class personal scripts

Users can add their own scripts with metadata, schedules, wake policies, reconcile, and doctor support through the official flow.
