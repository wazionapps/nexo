New in v7.35.0 — June 16, 2026 — MINOR

Cognitive OS Ola 4 — selective forget + recurring-incident diagnostic templates

Selective forget deletes a leaked secret or a genuinely-wrong memory and verifies zero survivors across every live store (all canonical databases, FTS, vector index, transcripts) with secure_delete=ON so the bytes are overwritten, or corrects a fact reversibly (soft, undoable). Recurring failure archetypes are distilled by Deep Sleep into reusable diagnostic templates that are primed before a matching action so the known checks run first; strong/weak marker tiers keep benign success phrasing (“exit 0 on success”) from ever seeding a template, and injection is guidance-only and never blocks. Closing a local-only followup-runner is no longer mis-flagged as an external real-world action. Builds on v7.34.0.

Builds on v7.33.0.
