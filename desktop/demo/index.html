A JavaScript simulation based on the current product

This page behaves like a short product film, but every scene is anchored to the current NEXO Desktop layout: active thread, archived conversations, in-thread search, quick search, preferences, tool cards, and the message composer.

What is simulated Real interface states, not fictional business modules: sidebar, archived, search, quick search, preferences, and tool output.
What is not simulated No live account, no writable sandbox, and no fake CRM or “client snapshot” surface that the current app does not expose.
Why this format It lets visitors see how the app feels without exposing a real environment or reducing the product to static screenshots.
States in the loop

Live thread

An active conversation with streaming-style replies, readable tool cards, status row, composer, export, and stop controls.

Archived conversations

The archived section opens with its own search field and restore flow, mirroring the actual structure in the sidebar.

Conversation search

The top bar shows the in-thread search mode with result count and next/previous navigation controls.

Quick search

A global conversation search overlay appears as its own state instead of pretending every workflow lives in one thread.

Preferences

The modal uses the real tab structure and NEXO Desktop preference fields, including theme, tool detail visibility, notification sound, and do-not-disturb.

Product honesty

The theater stays close to the existing product so the marketing page sells the real application rather than a future wishlist.
